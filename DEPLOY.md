# Развёртывание «Совет матерей» (прототип) на Coolify (Beget)

Это веб-приложение на **Next.js 16**. Оно общается с уже работающим
**self-hosted Supabase** (`https://supabase.sovetmam.ru`). Отдельная база данных
для самого приложения не нужна — все данные (меры поддержки и т. п.) уже в Supabase.

## Из чего состоит развёртывание

1. **GitHub** — здесь хранится код приложения. Coolify забирает код отсюда.
2. **Coolify** — собирает приложение из кода и запускает его на сервере Beget.
3. **Поддомен** (например, `app.sovetmam.ru`) — адрес, по которому открывается прототип.
4. **Переменные окружения** — секретные ключи. Их НЕ кладут в код, а задают
   в Coolify (см. ниже). В репозитории их нет (`.env.local` исключён через `.gitignore`).

---

## Переменные окружения (Coolify → Environment Variables)

| Имя | Значение | Обязательна |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://supabase.sovetmam.ru` (из `.env.local`) | да |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | публичный anon-ключ Supabase (из `.env.local`) | да |
| `SUPABASE_SERVICE_ROLE_KEY` | сервисный ключ Supabase (из `.env.local` / `MSP Supabase/.cursor/mcp.json`) | да |
| `ANTHROPIC_API_KEY` | — | нет (AI-разбор отложен) |

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY` — секрет с полным доступом к базе. Используется
> только на сервере (без префикса `NEXT_PUBLIC_`), в браузер не попадает. Не публиковать.
> Значения копировать из локального `.env.local` (его нет в репозитории).

---

## Настройки приложения в Coolify

- **Источник**: GitHub-репозиторий, ветка `main`.
- **Build Pack**: `Nixpacks` — Coolify сам определит Next.js и выполнит
  `npm install` → `npm run build` → `npm start`. Dockerfile не нужен.
- **Base Directory**: `/` (репозиторий = содержимое папки `sovetmam-web`).
- **Порт (Ports Exposes)**: `3000`.
- **Домен (Domains)**: `https://app.sovetmam.ru` — Coolify сам выпустит SSL
  (Let's Encrypt).

После «Deploy» Coolify скачает код, соберёт и запустит приложение.

---

## DNS (у регистратора домена sovetmam.ru)

Добавить запись типа **A** для поддомена на тот же сервер, где Coolify и Supabase:

```
app.sovetmam.ru  →  <IP-адрес сервера Beget>
```

После распространения DNS (от пары минут до пары часов) Coolify выпустит SSL.

## ⚠️ ВАЖНО: CORS на стороне Supabase

Браузерная часть приложения обращается к `https://supabase.sovetmam.ru` напрямую.
Нужно, чтобы self-hosted Supabase (шлюз **Kong**) разрешал источник
`https://app.sovetmam.ru`.

Признак проблемы: на сервере всё собралось, страница открывается, но **данные
(каталог мер) не подгружаются в браузере** — в консоли браузера ошибки CORS.
Тогда добавить `https://app.sovetmam.ru` в разрешённые origins в конфиге Supabase/Kong.

## Замечания про прототип

- Включён **демо-режим авторизации** (вход по кнопкам без пароля) — это нормально
  для показа клиенту. Перед боевым запуском убираем.
- Service worker (`/sw.js`) для установки PWA уже настроен.
- Меры поддержки уже загружены в Supabase — каталог наполнен сразу.

## Обновление приложения

Новый коммит в ветку `main` на GitHub → Coolify пересоберёт приложение
(автоматически при включённом auto-deploy или по кнопке «Deploy»).
